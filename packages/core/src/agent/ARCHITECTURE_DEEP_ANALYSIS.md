# Agent 架构深度分析

本文档深度分析 `/packages/core/src/agent` 目录的架构设计，包含详细的流程图说明。

## 目录结构概览

```
agent/
├── agent/                    # 核心代理运行时
│   ├── index.ts             # StatelessAgent 主入口
│   ├── run-loop.ts          # 主运行循环
│   ├── run-loop-stages.ts   # LLM/Tool 阶段执行
│   ├── runtime-composition.ts # 运行时组装
│   ├── llm-stream-runtime.ts # LLM 流处理
│   ├── tool-runtime*.ts     # 工具运行时
│   ├── compaction*.ts       # 消息压缩
│   ├── timeout-budget.ts    # 超时预算管理
│   └── error*.ts            # 错误处理
├── tool-v2/                  # 工具系统 v2
│   ├── orchestrator.ts      # 工具编排器
│   ├── router.ts            # 工具路由
│   ├── registry.ts          # 工具注册表
│   ├── contracts.ts         # 类型契约
│   ├── context.ts           # 执行上下文
│   ├── permissions.ts       # 权限管理
│   ├── handlers/            # 工具处理器
│   ├── runtimes/            # Shell 运行时
│   ├── task-*.ts            # 任务系统
│   └── skill/               # 技能系统
├── auth/                     # 认证授权
│   └── contracts.ts         # 授权契约
├── app/                      # 应用层服务
│   └── agent-app-service.ts # Agent 应用服务
└── types.ts                  # 公共类型定义
```

## 二、主运行循环流程图

### 2.1 RunLoop 核心流程

`run-loop.ts` 是整个代理的核心执行引擎：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           runAgentLoop() 主循环                              │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
                    ┌─────────────────────────────────┐
                    │      初始化状态                   │
                    │  stepIndex=0, retryCount=0      │
                    │  runOutcome='done'              │
                    └─────────────────────────────────┘
                                      │
                                      ▼
                    ┌─────────────────────────────────┐
                    │   while (stepIndex < maxSteps)  │◄─────────────────┐    │
                    └─────────────────────────────────┘                  │    │
                                      │                                  │    │
                                      ▼                                  │    │
                    ┌─────────────────────────────────┐                  │    │
                    │   resolvePreStepTerminalState   │                  │    │
                    │   (检查终止条件)                  │                  │    │
                    └─────────────────────────────────┘                  │    │
                                      │                                  │    │
                    ┌─────────────────┴─────────────────┐                │    │
                    ▼                                   ▼                │    │
            ┌──────────────┐                    ┌──────────────┐        │    │
            │ 终止状态已确定 │                    │ 继续执行步骤  │        │    │
            │ break 循环    │                    │ stepIndex++  │        │    │
            └──────────────┘                    └──────────────┘        │    │
                                                      │                  │    │
                                                      ▼                  │    │
                                    ┌─────────────────────────────────┐  │    │
                                    │     prepareMessagesForStep      │  │    │
                                    │     (消息准备与压缩)              │  │    │
                                    └─────────────────────────────────┘  │    │
                                                      │                  │    │

                                    │         runLLMStage             │  │    │
                                    │     (LLM 调用阶段)               │  │    │

                                                      │                  │    │

                    │      runToolStage           │     │  检查用户消息         │

                    ┌─────────────────────────────────┐                       │
```

## 总结

该 Agent 架构采用了清晰的分层设计和模块化组织：

1. **核心运行时** (`agent/`): 负责主循环、LLM 交互、工具调度
2. **工具系统** (`tool-v2/`): 提供可扩展的工具框架和权限控制
3. **授权系统** (`auth/`): 实现细粒度的权限和审批机制
4. **任务系统**: 支持任务编排和子代理管理

关键设计原则：
- 无状态设计，支持水平扩展
- 依赖注入，便于测试和替换
- 事件驱动，支持异步流处理
- 分层超时，保护系统稳定性
- 细粒度权限，确保安全可控

```typescript
interface StatelessAgent {

// 运行时契约
interface RunLoopRuntime {

// 工具执行上下文
interface ToolExecutionContext {
  workingDirectory: string;

  sessionState: ToolSessionState;
  authorization: ToolAuthorizationContext;
  onEvent?: (event: ToolExecutionEvent) => void;

// 工具处理器
interface ToolHandler<TArgs = unknown> {
  readonly spec: ToolSpec;
  parseArguments(raw: string): TArgs;
  plan(args: TArgs, ctx: ToolExecutionContext): ToolExecutionPlan;
  execute(args: TArgs, ctx: ToolExecutionContext): Promise<ToolHandlerResult>;
}

// 授权服务
interface ToolAuthorizationService {
  authorizeExecution(request: AuthorizationExecutionRequest): Promise<AuthorizationExecutionResult>;
  requestPermissions(request: ExplicitPermissionGrantRequest): Promise<ToolPermissionGrant>;
}

// 任务记录
interface TaskRecord {
  id: string;

  status: TaskStatus;
  priority: TaskPriority;
  blockedBy: string[];
  blocks: string[];
  progress: number;

}
```

## 二十、架构演进建议

### 20.1 当前架构优势

- **无状态设计**: StatelessAgent 不持有运行状态，支持水平扩展
- **依赖注入**: 通过工厂函数组装运行时，便于测试和替换
- **事件驱动**: 流式处理支持实时响应和增量持久化
- **分层超时**: 执行级和阶段级超时保护系统稳定性
- **细粒度权限**: 文件系统、网络、审批多层权限控制

### 20.2 潜在改进方向

1. **可观测性增强**
   - 增加分布式追踪集成
   - 完善指标采集和告警

2. **容错能力**
   - 增加断路器模式
   - 实现优雅降级策略

3. **性能优化**
   - 工具执行结果缓存
   - 消息压缩策略优化

4. **安全加固**
   - 敏感数据脱敏
   - 审计日志完善
